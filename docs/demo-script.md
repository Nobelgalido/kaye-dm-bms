# Demo Script (~45 seconds)

One coherent story: a wave hits, the system keeps up, the owner sees the business. Numbered so recording the GIF is mechanical — each step is one screen action.

Seeded logins: `owner@kayedm.local` / `cashier@kayedm.local`, both `KayeDM#2026`.

1. **Log in as Cashier** (`cashier@kayedm.local`) — lands on `/pos` automatically; the sidebar shows only what a cashier needs (POS, Arrivals, Production, Waste).
2. **Bus Arrivals** (sidebar → Buses → Arrivals): quick-log the arrival — company **DLTB**, bus **8112**, route **Manila → Sorsogon**. Timestamps now and starts the wave clock; the trip appears on today's board immediately.
3. **Back to POS**: rapid-fire six passenger orders in under a minute — category tabs, big-button menu grid, Cash/GCash quick-tender buttons, instant change calculation on Complete.
4. **Crew meal**: toggle Crew Meal Mode — the ticket header flips to a yellow **"CREW MEAL"** banner — select the DLTB trip and Driver role, Complete. A ₱0 order is logged and credited against DLTB's monthly allowance.
5. **Log out, log in as Owner** (`owner@kayedm.local`) — lands on `/dashboard` automatically.
6. **Log the day's market-run expense** (sidebar → Expenses → Entry) — one quick entry, then back to the dashboard.
7. **Dashboard**: point at today's sales-by-hour spike lining up with the bus-arrival marker, then the 30-day revenue vs. expenses vs. net trend just below it.
8. **Reports** (sidebar → Reports — one group, four reports): open the **Crew Meals** report, filter to DLTB this month, then the **Expenses** report for the same range.
9. **Daily Closing** (sidebar → Sales → Daily Closing): review today's sales, expenses, and net in one glance, then **Confirm — Close Today** to lock the snapshot. Orders, voids, and expenses on or before today are now immutable.
